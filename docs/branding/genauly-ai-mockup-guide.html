1 · The brief

Paste this block at the top of any chat-based tool (ChatGPT, Gemini, v0) or use it as the description in an image tool. It carries the product, the feeling, the exact palette, and the hard rules.

Master brief · paste firstGenauly is a mobile-first web app that helps adults break through the B1–B2 German "plateau" and build real-life fluency for the workplace and everyday admin (Behörde, bank, doctor, housing), plus exam prep (telc B2 Beruf, Goethe B2). Audience: adults 25–45 learning German as immigrants and professionals. Brand feeling: encouraging, modern, precise, a little playful. Confident but warm. NOT childish, NOT corporate-SaaS, NOT a generic language-app mascot. Design language: - FLAT color. NO gradients anywhere (not on the logo, not on buttons, not on backgrounds). - Bold and colorful, with crisp rounded shapes and generous spacing. - Solid-color fills, clean 2px line icons, strong type hierarchy. Exact palette (use only these): - Cobalt #2B4FE0 primary / brand / buttons - Butter #FFC24D warm highlight - Coral #FF6B54 playful accent - Sky #CBD8FF soft fill / chips - Ink #111318 text - Cream #F3F0E7 background (Keep a success-green and error-red only for status, used sparingly.) Avoid: gradients, glossy 3D, bevels/embossing, drop-shadow skeuomorphism, generic SaaS purple, a rounded-square gradient "G" that looks like the Canva logo, stock blobs, neon cyberpunk, lorem ipsum text.
The name comes from genau ("exactly"). If you want the mark to say something, precision / exactness / "the click of getting it right" is the core idea. A German umlaut (two dots) is a distinctive, ownable device if you want to lean German.

7 · Restyle your real screenshots

If you want to keep your existing layouts and only explore color, feed a screenshot as reference. Best in ChatGPT (GPT-4o image), Gemini, or Krea (image-to-image). Upload the screenshot, then:

ChatGPT / Gemini · with a screenshot attachedHere is a screenshot of my app screen. Recolor it to this flat palette, keeping the exact same layout, icons, spacing and text: cobalt #2B4FE0 for primary buttons and the active state, cream #F3F0E7 background, butter #FFC24D and coral #FF6B54 as accents, ink #111318 text, sky #CBD8FF for chips. Make all buttons a solid flat color (no gradient). Do not redesign anything, only change the colors. Return the same screen restyled.
Reality check: image tools will redraw the screen and often drift from your real layout (fonts, spacing). They're great for a color vibe, not pixel-accurate specs. For a pixel-accurate recolor of the real app, just tell me the hex codes you landed on and I'll apply them to the live app in minutes (that path is exact).

You can export fresh screenshots of any current screen to feed these tools; ask and I'll generate clean PNGs of any page (light or dark) for you to upload.

8 · Pro tips & settings

Sizes to request

  • App icon: 1024 × 1024 px, square.
  • Wordmark: wide, ~3:1 (e.g. 1800 × 600).
  • Mobile screen: 390 × 844 (or 1290 × 2796 for @3x retina).
  • Moodboard: 16:9.

Getting flat, on-brand results

  • Always include the words "flat, solid color, no gradient, no shadow, no bevel". Models love gradients by default.
  • Paste exact hex codes every time. Without them you'll get "close-ish" colors.
  • In Midjourney add --style raw (less decorative) and a --no gradient, 3d, glossy, canva negative.
  • In Recraft, set a brand palette and pick the "Flat illustration" / "Icon" style so every generation stays on-palette and exportable as SVG.
  • Generate 4–6 variations, choose one, then refine with "keep everything, change only X".

Keeping your layout

  • For screens you designed, upload a screenshot as a reference/structure image and say "match this layout".
  • Or skip the redraw entirely: choose colors here, and I apply them to the real app so nothing about your layout changes.

Avoiding the "AI look"

  • No generic SaaS purple, no glowing gradients, no 3D blobs, no mascot unless you want one.
  • One dominant color (cobalt) + one or two accents beats a rainbow. Let whitespace do work.

9 · Handing results back to build

Once you've generated things you love, give me the following and I'll implement it in the real app (tokens, logo assets, favicons, PWA icons), no redesign of your layouts:

  • Final palette: the hex codes you settled on (even if they differ from this guide).
  • Logo: the chosen mark as SVG if possible (Recraft/Ideogram export it), otherwise a high-res PNG, plus which color combo for the small favicon.
  • Screens: either the image you like (as a visual target) or the v0.dev link / code — code is ideal because I can adapt it directly.
  • Type: the display font name if you picked one.
The exact, reversible path: the app's colors live in a few design-token variables (src/index.css + tailwind.config.ts, light + dark). Once you give me final hex codes and a logo, I wire them in, regenerate the logo / favicons / PWA icons, run the build, and show you the real app. Everything stays flat and gradient-free, and every screen re-themes at once.
