## Gaps found

1. **🔴 Latent eager-bundle landmine (Heute).** `Dashboard.tsx` eagerly imports
   `intentCardsForMode` from `intentCards.ts`, which statically imported `filterVocab` from the
   245 kB `@/data/vocabulary` bank. It stayed out of the main chunk only because the two functions
   using it (`cardMeta`, `cefrRange`) were dead code, so Rollup tree-shook them. The moment anyone
   rendered `cardMeta` on a Heute card (its intended use), the whole bank would land in the eager
   chunk and blow the ~78 kB Dashboard invariant CLAUDE.md protects. **→ Fixed (see below).**
2. **🟠 No interactive "claim" moment on the Can-Do quest card.** The plan promised a "claim moment
   via `canDo.ts` thresholds"; the code flipped milestones to "achieved" passively with no claim
   interaction or reward. **→ Fixed (see below).**
3. **🟡 Onboarding defers name / exam-date / rhythm — resolved as acceptable.** These are all
   editable in `Settings.tsx` (name L128, XP goal L149, exam date L165), so no user is stuck without
   them. The plan's "collect on the first end screen" contextual prompts were never built, but
   Settings covers it. No code change needed.
4. **🟡 Word-order quiz blocks feed FSRS without a `latencyMs` sample** (`QuestionViews.tsx`
   `WordOrderView`). Pre-existing, not a redesign regression; `reviewVocab` tolerates `undefined`.
   Soft signal loss on one graded path. Left as-is (out of redesign scope).
5. **🟡 Two of six city buildings can never light up.** `bank` and `wohnhaus` have empty
   `domains`/`themeIds` in `DOMAIN_BUILDINGS` (intentional — await future banking/housing content).
   The city reads as partly inert until those packs ship. Noted, no action.

Minor: `<Gloss>` reflows horizontally on toggle (no reserved min-width); recharts loads with the
analytics chunk even when Details is collapsed (not a budget issue — lazy route). Both cosmetic.

## Actions taken (this session)

1. **De-risked the eager bundle (gap #1).** Removed the dead `cardMeta`/`cefrRange` and the
   `filterVocab`/`CEFR_ORDER` imports from `src/features/dashboard/intentCards.ts`, so the eager
   Dashboard path no longer has any static edge into a content bank — the invariant is now
   structural, not an accident of tree-shaking. Main chunk unchanged (79.1 kB).
2. **Built the quest claim moment (gap #2).** Added persisted `claimedMilestones: string[]` +
   `claimMilestone(id)` to `useSettingsStore` (rides cloudSync via the settings jsonb blob; no
   version bump needed — new field merges to its default). Fortschritt now shows a reward-gold,
   spring-in "Quest geschafft" card for any achieved-but-unclaimed milestone with an "Einlösen"
   button that advances to the next unclaimed win. Reduced-motion honored via the global
   `MotionConfig`. Added a `claimMilestone` idempotency unit test (`tests/stores.test.ts`, 63 tests).

All gates green after the changes.
